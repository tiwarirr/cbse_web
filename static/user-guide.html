CBSE Result Dashboard
User Guide

Parsing, analysing, and exporting CBSE board exam results from gazette TXT files.

Overview

The CBSE Result Dashboard is a web application that parses CBSE board result gazette files and provides detailed analytics. It supports both Class X (Secondary School Examination) and Class XII (Senior School Certificate Examination) gazette TXT files.

The app runs in two modes:

  • API mode — connected to a Flask backend with SQLite. Data is persisted on the server, and users can log in for multi-user access.
  • Local mode — operates entirely in the browser using localStorage. No server required; everything runs client-side.
Note: The app auto-detects which mode to use. If a Flask server is detected, it uses the API; otherwise, it falls back to localStorage.

Quick Start

  1. Upload gazette TXT files. Drag-and-drop or click to browse for Class X and/or Class XII files on the upload screen. The app auto-detects the class from the file content.
  2. Click "Analyse Results". The app parses the gazette and loads the dashboard.
  3. Explore the dashboard tabs. Navigate through Summary, Subjects, Merit List, Gender, All Students, Section Review, Teacher Review, and Follow-Up.
  4. Upload master data (optional). Upload Excel files for Student Master, Teacher Mapping, or Internal/Practical Marks to enrich the analysis.
  5. Export to Excel. Click the "Export Excel" button to download a formatted workbook.

First-Time Login

  1. Open the app. If running in API mode, the login screen appears.
  2. Sign in with the admin credentials, or create a new account in the "New user?" section.
  3. Regular users can create their own accounts via the sign-up form.

User Management (Admin only)

Click "Manage Users" in the header to open the user management modal. From here you can:

  • Create new user accounts (admin or regular user).
  • Toggle premium features per user: Student Master upload, Teacher Mapping upload, Performance Marks upload, Excel Export.
  • Delete users (except the master admin account or yourself).

Change Password

Click "Change Password" in the header to update your password. Minimum 4 characters.

Working with Sessions

A session represents a single school's result data for a given year, containing Class X and/or Class XII gazette data plus any associated master data (student lists, teacher mappings, performance marks, follow-up notes).

Uploading Result Files

  1. Click "Add School" or navigate to the upload screen.
  2. Drag-and-drop TXT files onto the Class X and/or Class XII cards, or click to browse.
  3. The app auto-detects whether the file is Class X or Class XII by checking headers like "SECONDARY SCHOOL EXAMINATION" or "SENIOR SCHOOL CERTIFICATE".
  4. Once files are loaded, click "Analyse Results" to parse and load the dashboard.

Session Limitations

Regular users can upload a maximum of 2 result files total. Admin users have no limit.

Multi-School Workspace

You can work with multiple schools simultaneously:

  • Select schools in the workspace panel by clicking "Select" on school cards.
  • Switch to "Multi-School" view mode for cross-school comparisons.
  • View the Summary tab to see each school's stats side-by-side.
  • The Merit List tab combines students across schools into a single ranked list.
  • Save combinations to quickly restore a set of selected schools later.

Backup & Restore

  • Click "Backup" to download a JSON file containing all your sessions and combinations.
  • Click "Restore Backup" and select a JSON backup file to restore data.
  • In local mode, data persists in the browser's localStorage until cleared.
  • Click "Clear Saved" to remove all sessions from storage.

Master Data Enrichment

Master data is uploaded via Excel files (.xlsx, .xls, or .csv) and enriches the parsed student records with section, class teacher, subject teacher, and internal/practical marks information.

Premium features: Uploading master data requires the appropriate feature permissions. Admins have these by default; regular users need an admin to enable them.

Performance Marks (Internal / Practical)

Upload internal assessment or practical marks. These are used to derive theory marks from the gazette total marks.

  • Select "Internal Marks" or "Practical Marks" from the dropdown.
  • Upload an Excel file with columns for rollNo, class, subjectCode, componentMarks, and componentMaxMarks.
  • The app automatically derives theory marks (total − component) for enriched analysis.
  • A rich upload report shows matched, missing, and duplicate rows.

Dashboard Tabs

Summary

Single & Multi-School

The summary tab provides a high-level overview:

  • Result stripe — class badge, exam name, school info, and pass rate.
  • Stat cards — total students, passed, compartment, absent, boys/girls breakdown.
  • School topper card — top student with best-of-5-with-English score.
  • Charts — result distribution (donut), boys vs girls pass % (bar), grade distribution (bar).
  • Subject-wise breakdown table — per-subject pass/fail/absent counts.

In multi-school mode, a comparison grid shows all schools side-by-side with topper info, core metrics table, and subject averages table.

Subjects

Single School

Detailed subject performance analysis:

  • Analysis mode selector — switch between Total Marks, Theory Marks, or Internal/Practical Marks (when performance data is uploaded).
  • Average marks chart — horizontal bar chart of subject averages.
  • Marks distribution heat map — colour-coded grid showing percentage of students in each marks bucket per subject.
  • Detailed breakdown table — total, absent, NA, applicable, failing, threshold %, average, max/min, and marks distribution per subject.
  • Teacher detail toggle — expands each subject row to show teacher-level breakdowns with performance metrics per section.

Merit List

Single & Multi-School

Ranked list of PASS and COMP students with configurable scoring:

  • Scoring modes:
    • Best N with English — top N subjects including English (default).
    • Best N (any subjects) — top N subjects regardless of subject.
    • All Subjects — percentage of total marks across all subjects.
  • N count — number of subjects to consider (default 5).
  • English subject code — customise which subject code counts as English.
  • Gender filter — all, boys only, or girls only.
  • Show count — top 10, 25, 50, or all.
  • Dense ranking — ties share the same rank number.
  • Visual indicators — gold/silver/bronze for top 3, used subjects highlighted in marks display.

In multi-school mode, students across selected schools are pooled into a single combined merit list with school and class labels.

Gender

Single School

Comparative analysis between boys and girls:

  • Overall comparison cards showing student counts and pass percentages.
  • Pass % bar chart — boys vs girls.
  • Average total marks bar chart.
  • Subject-wise average comparison chart (boys vs girls).
  • Grade distribution bar charts for boys and girls separately.

All Students

Single School

Searchable, filterable, sortable table of all students:

  • Search — type to filter by name or roll number.
  • Result filter — all, pass, compartment, absent, or fail.
  • Gender filter — all, boys, or girls.
  • Sortable columns — click any column header to sort (roll no, name, result, total, %, individual subjects).
  • "Export Excel" button — downloads the current filtered and sorted view as a formatted Excel file.

Section Review

Requires Student Master upload

Board-result review grouped by section and class teacher:

  • Sections listed with their class teachers.
  • Per-section stats: total students, pass %, compartment count, fail, absent.
  • Average total marks and section topper.
  • Weakest subjects identified per section.
  • All columns are sortable.

Teacher Review

Requires Teacher Mapping upload

Board-result review by subject teacher across sections:

  • Analysis mode selector (Total / Theory / Component marks).
  • Teacher, department, subject, section, taught count.
  • Threshold %, average, school average, variance.
  • Distinction and low-performer counts.
  • All columns are sortable.

Follow-Up

Single School

Post-result tracking for compartment, fail, and absent students:

  • Auto-lists students needing attention (COMP, Grade E subjects).
  • For each student: weak subjects identified.
  • Status dropdown — Pending, Reviewed, Parent Meeting Done, Supplementary Prep Started, Closed.
  • Owner — assign a staff member.
  • Remarks — free-text notes.
  • All changes are auto-saved in real-time to the server or localStorage.

For the merit list, the default scoring method is "Best N with English" (best 5 subjects including English), matching CBSE's standard merit ranking approach. The scoring is fully configurable via the settings panel on the Merit tab.

Performance Analysis Modes

When performance marks have been uploaded, the Subjects and Teacher Review tabs offer three analysis modes:

  • Total Marks — the gazette total out of 100 (no enrichment).
  • Theory Marks — derived as total − component marks.
  • Internal / Practical Marks — the uploaded component marks.

When theory or component modes are active, all pass thresholds, averages, and distributions use those marks instead of the total.

Excel Export

Single & Multi-School Premium feature

Click "Export Excel" to download a formatted workbook. In single-school mode, the workbook contains:

  • Summary — overall results, gender summary, grade distribution, subject-wise breakdown (with colour-coded pass %).
  • Class X / XII Subjects — subject-wise analysis with appeared/pass/fail, averages, and grade counts.
  • Class X / XII Merit — all PASS students ranked by total marks with subject marks and grades (top 3 highlighted with medal colours).
  • Gender Analysis — boys vs girls comparison tables and grade distributions.

In multi-school mode, the export includes:

  • Comparison — school-by-school metrics.
  • Subject Averages — per-subject averages across schools.
  • Combined Merit — cross-school merit list (when viewed).

From the All Students tab, the "Export Excel" button exports the currently filtered/sorted view as it appears on screen.

Gazette File Format

The app expects CBSE gazette TXT files with the following layout:

Class X Format

Each student record spans two lines:

  • Line 1: ROLLNO GENDER NAME SUBJECT_CODES RESULT
    Example: 12345678 M STUDENT NAME 184 041 086 087 402 PASS
  • Line 2: MARKS_OR_GRADES
    Example: 89 95 92 78 85

Result can be PASS, COMP (compartment), ABST (absent), or FAIL.

Class XII Format

Similar two-line structure but with grade codes between subject codes and result:

  • Line 1: ROLLNO GENDER NAME SUBJECT_CODES GRADES RESULT
    Example: 87654321 F STUDENT NAME 301 042 043 044 055 A1 B2 A2 B1 A1 PASS
  • Line 2: MARKS
    Example: 95 78 88 92 96

The app gracefully handles files without explicit grade codes by inferring grades from marks.

Auto-Detection

The app detects the class from the file header text:

  • Class X: header contains "SECONDARY SCHOOL EXAMINATION"
  • Class XII: header contains "SENIOR SCHOOL CERTIFICATE"

Tips & Shortcuts

  • Drag & drop TXT files directly onto the upload cards.
  • Use Enter in password fields on the login screen to submit.
  • Mapping Excel files placed in the static/mappings/ folder are auto-loaded when a session is opened (named SCHOOLCODE-YEAR-student_master.xlsx or SCHOOLCODE-YEAR-teacher_mapping.xlsx).
  • The storage indicator in the workspace panel shows how much localStorage space is used.
  • Use Backup regularly to export all your sessions as a JSON file for safekeeping.
  • In multi-school mode, the "Mixed Year" banner warns when selected schools span different exam years.
